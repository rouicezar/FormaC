# 私有知识库客服系统需求

## 1. 产品目标

构建一个面向匿名访客、外部用户和内部员工的知识库查询、自然语言问答与客服一体化系统。知识文件保留在管理员授权的本地目录中，系统通过手动扫描和每隔一天自动扫描维护索引，并通过独立网页与飞书机器人提供原文查询和带引用的知识问答。

## 2. 用户与权限

- 管理员：配置知识目录、模型、飞书、扫描计划、用户身份与隐私策略，并查看全局查询、问答次数及记录。
- 匿名用户：未绑定账号也可在 Web 查询和问答公开知识，历史仅保存在浏览器本地。
- 外部用户：飞书绑定成功后默认身份；只能访问公开知识，可在 Web 与飞书使用并同步个人历史。
- 内部员工：只有管理员明确授权后才能形成；可访问公开和敏感知识。
- 管理员未授权前不得把用户视为内部员工；身份识别失败时必须按外部用户处理。
- 网页端支持飞书绑定；保留本地超级管理员应急入口。匿名历史合并必须得到用户明确确认。

## 3. 知识来源与更新

- 管理员配置一个本地知识根目录，包含 `public/` 和 `sensitive/` 两个固定子目录。
- 不提供文件上传，不复制原始文件，只读取授权目录。
- 首版支持 PDF、DOCX、Markdown、TXT、XLSX 和 PPTX。
- 支持管理员手动扫描和每隔一天自动增量扫描。
- 使用路径、修改时间、大小和内容哈希识别新增、修改与删除。
- 扫描必须生成新增、更新、删除、失败和跳过统计；单文件失败不得中断整批任务。
- 新索引成功写入前保留旧索引，避免解析失败造成知识突然不可用。

## 4. 原文查询与知识问答

- 使用本地 Embedding、关键词索引、混合检索和本地重排。
- 原文查询与知识问答必须是两个独立入口、独立记录类型和独立统计指标。
- 原文查询只检索原始资料，不调用大模型，返回原文片段、文件名及页码、工作表、幻灯片或文本位置。
- 原文查询使用独立 `/search` API 和独立服务，不得通过 `/ask`、模型适配器或提示词模拟原文查询。
- 知识问答必须先检索知识库，再由获准使用的模型基于证据总结或汇总，并展示同一证据包的引用。
- 支持多轮问答、流式输出、停止生成、复制、重新生成和回答反馈。
- 管理员查询测试台可查看召回片段、分数、重排结果和实际模型上下文。
- 证据不足时不得编造答案，应说明缺口并触发人工接管。

## 5. 模型与隐私

- 模型提供商必须可插拔，支持 OpenAI 兼容云端模型（首个为 DeepSeek）和本地模型（首个适配 Ollama）。
- API Key 不得进入源码、日志或 Git，只能保存在本地秘密配置中。
- 公开知识可提交给当前选择的模型生成答案。
- 敏感知识仅内部员工可检索。
- 管理端提供“允许敏感内容发送给云端模型”选择框，默认关闭。
- 开关关闭且选择云端模型时，敏感查询只能返回原文检索结果，不得把问题相关敏感片段发送到云端。
- 使用本地模型时，可基于敏感内容生成回答。
- 敏感开关变更必须二次确认并写入审计日志。

## 6. 客服与飞书

- 支持飞书私聊和群聊 `@机器人` 问答。
- 飞书回调必须完成签名验证、事件去重和异步回复。
- 飞书采用纯文本输入和输出，不设计菜单或消息卡片。
- 支持 `查询：问题`、`问答：问题`、`历史`、`帮助`；无前缀文本默认按知识问答处理。
- 回答包含结论或原文、引用来源及必要的权限和隐私提示。
- 三类情况触发人工接管：用户明确要求、证据不足或低置信度、投诉/退款/合同/隐私等高风险主题。
- 工单应包含用户身份、原问题、会话摘要、机器人回答、检索证据和触发原因。
- 客服接管后机器人暂停自动回答，直到客服明确恢复。

## 7. 前端范围

### 管理后台

- 管理首页、知识库与扫描详情、模型配置、用户授权、全局记录、数据分析、快捷入口管理、飞书接入、审计日志和检索实验室。

### 独立网页问答端

- 用户首页、原文查询、知识问答、快捷入口、历史记录、只读文档中心、飞书绑定和个人中心。
- 用户可查看自己的查询与问答次数及历史；快捷入口结合个人频次与管理员配置生成。
- 桌面端优先，同时适配手机浏览器。

### 飞书端

- 只使用纯文本完成查询、问答、历史和帮助；不使用菜单或卡片。

### 简体中文界面

- 所有面向用户和管理员的 Web 前端必须完全使用简体中文。
- 页面标题、导航、按钮、表单标签、占位提示、加载状态、空状态、确认框、错误提示、引用说明和无障碍标签均不得遗留英文界面文案。
- 文档原文、文件名、模型正式名称、协议名和必要技术标识可以保留其原始语言。
- HTML 文档语言必须声明为 `zh-CN`，并以中文界面回归检查作为前端验收门禁。

## 8. 非目标（首版）

- 图片和扫描版 PDF 的 OCR。
- 企业微信、网页访客客服插件和语音客服。
- 自动修改或删除原始知识文件。
- 无引用的通用聊天机器人能力。

## 9. 验收标准

- 所有用户端和管理端页面通过简体中文界面回归检查，不包含模板遗留英文文案。
- 外部客户在网页和飞书中均无法召回或间接获得敏感内容。
- 敏感云端开关关闭时，测试桩证明敏感片段未进入云端模型请求。
- 六类文档均有解析与引用测试样本。
- 新增、修改和删除文件后，手动及每日扫描能正确更新索引。
- 固定问答评测集可验证召回、引用正确性和拒答行为。
- 三种人工接管触发均可形成工单，接管期间机器人不会抢答。
- 飞书重复事件不会产生重复回答。
