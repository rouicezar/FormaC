# 最小闭环手动验收指南

## 1. 启动

```bash
cd advanced_llm_apps/private_knowledge_customer_service
./scripts/start.sh
```

保持该终端打开，访问 <http://127.0.0.1:5177>。页面顶部应显示“后端已连接”。

## 2. 首次扫描

点击“立即扫描知识目录”。默认合成知识目录应显示：

- 新增：2
- 失败：0
- 引用来源可出现 `public/退款政策.md` 和 `sensitive/内部折扣规则.md`

## 3. 公开问答和引用

保持默认设置：

- 测试身份：外部客户
- 回答模型：Ollama 本地模型

问题输入“客户申请退款的期限是多久？”，点击“发送问题”。预期：

- 回答包含“七个自然日”。
- 引用只来自 `public/退款政策.md`。
- 引用显示文件名和行号。

再输入“内部折扣底线是什么？”。预期引用仍全部为公开分区，不得出现 `sensitive/`。

## 4. 敏感原文模式

选择：

- 测试身份：内部员工
- 回答模型：DeepSeek 云端模型
- “允许敏感原文发送到云端”：保持关闭

输入“内部折扣底线是什么？”。预期：

- 页面显示“原文模式”。
- 回答包含“八五折”。
- 引用包含 `sensitive/内部折扣规则.md` 和行号。
- 即使没有配置 DeepSeek API Key，也能返回原文，因为该请求不应调用云端。

## 5. 新增、修改和删除

在授权目录的 `public/` 中新增 `验收.txt`：

```text
验收代号为青松一号。
```

点击扫描，预期“新增 1”；公开身份提问“验收代号是什么？”，应回答“青松一号”并引用该文件。

把文件改为：

```text
验收代号已更新为青松二号。
```

再次扫描，预期“更新 1”；问答只应引用“青松二号”，不能继续返回旧内容。

删除该文件并再次扫描，预期“删除 1”；后续引用不得再出现该文件。

## 6. 重启持久化

另开终端执行：

```bash
./scripts/stop.sh
./scripts/start.sh
```

再次点击扫描。未修改默认样本文档时，预期“未变化 2”，公开和敏感问答仍能返回之前的引用。这证明业务元数据和两套 RAGLite 索引均保存在 PostgreSQL 数据卷中。

## 7. 停止

```bash
./scripts/stop.sh
```

预期提示“数据卷已保留”。如需诊断，运行 `./scripts/status.sh` 查看前后端最近日志和数据库容器状态。
